Ручная проверка после правок пайплайна оружия v3 (reload / interact / HUD).

## Подготовка

- Пересобрать compendium: закрыть Foundry → `npm run pack:sh-test-items` и `npm run pack:sh-test-actors`.
- Импортировать актёров WV3 из `sh-test-actors` (Shooter, Pistol, Auto, Bolt и т.д.).

## ТП-9 «Шквал» (внешний магазин)

1. Взять пистолет в руки, **без магазина**: HUD показывает «нет магазина», не `0+0/12` как установленный пустой.
2. **Взаимодействовать → Перезарядить**: выбор совместимого магазина; при пустом патроннике в chain/перезарядке добавляется **Затвор**.
3. С патроном в патроннике: смена магазина **не трогает** патронник и **не** добавляет Затвор.
4. ПКМ по предмету в инвентаре / «в руках» / HUD — то же меню **Взаимодействовать**.

## ТВ-5 «Рубеж» (внутренний магазин, loadAmount = 5)

1. Пустой резерв: **Зарядить X** добавляет 5 патронов за стоимость loadX AP.
2. При полном резерве **Зарядить X** / **Зарядить 1** недоступны.
3. **Перезарядить** дозаряжает до ёмкости (5).
4. **Опустошить**: резерв сбрасывается, затем **Затвор** очищает патронник (если был заряжен).

## HD2-like (временно: в листе оружия capacity = 10, loadAmount = 5)

1. Два **Зарядить X** → 10 патронов; третье действие недоступно.

## ТА-7 / burst-auto

1. Режим очереди/авто: при нехватке патронов на середине серии **лишние AP не списываются** после последнего возможного выстрела.

## Multi-block (лазер: снаряд + батарея)

1. HUD показывает счётчики **всех** блоков активной линии.
2. Выстрел при провале второго блока **не** расходует первый (rollback).

## Магазины

1. `/N` в счётчике берётся из **capacity предмета-магазина**; fallback — capacity блока на оружии.
2. Загрузка не даёт переполнить nested storage магазина.
