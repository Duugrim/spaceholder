# План: Боеприпас-Контейнер как магазин

## Что меняем
- В [`module/sheets/item-sheet.mjs`](E:/FoundryVTT/Data/systems/spaceholder/module/sheets/item-sheet.mjs) добавить режим `containerUsesNestedStorage`, когда у предмета включены оба тега: `system.itemTags.isAmmo` и `system.itemTags.isContainer`.
- Для этого режима строить данные вкладки “Контейнер” из `system.storage.contents`, потому что оно уже нормализуется в листе предмета и уже отображается в инвентаре персонажа через nested-storage snapshots.
- Обычные контейнеры без тега `Боеприпас` оставить на текущем `system.container` / `containerHostId` поведении.
- Добавить обычным контейнерам общие лимиты: максимум предметов и/или максимум веса. Для гибрида `Боеприпас` + `Контейнер` использовать тот же функционал лимита количества, но значение брать из `system.weapon.ammo.capacity`, то есть из поля “Размер магазина” во вкладке “Боеприпас”.

Ключевая текущая развилка:

```javascript
system.storage = normalizeNestedStorage(system.storage);
if (system.itemTags.isContainer) {
  const panel = await this._buildContainerPanelContext(actor);
  context.containerGear = panel.containerGear;
}
```

## UI вкладки “Контейнер”
- В [`templates/item/parts/item-container.hbs`](E:/FoundryVTT/Data/systems/spaceholder/templates/item/parts/item-container.hbs) добавить ветку `{{#if containerUsesNestedStorage}}` перед текущими `world/actor` режимами.
- В этой ветке показать список вложенных патронов из `system.storage.contents`: иконка, имя, количество, вес, бейдж “вложенное хранилище”, кнопки “извлечь в инвентарь” и “удалить”.
- Добавить drop-зону для перетаскивания предметов в магазин. Для предмета на актёре это будет перенос с уменьшением/удалением исходного stack-а; для world/sidebar item можно сделать snapshot без удаления источника.
- Ограничить drop предметами с тегом `Боеприпас`; магазины-патроны с включённым `connector.enabled` лучше не принимать как “снаряды” по умолчанию, чтобы не получить магазин в магазине случайно.
- Для обычного контейнера добавить настройки лимитов на этой же вкладке: включаемый/пустой лимит количества и включаемый/пустой лимит веса. Для гибридного магазина эти поля количества не дублировать: показывать “занято / размер магазина” и ссылаться на настройку во вкладке “Боеприпас”.

## Логика хранения
- Переиспользовать [`module/helpers/item-nested-storage.mjs`](E:/FoundryVTT/Data/systems/spaceholder/module/helpers/item-nested-storage.mjs): `getNestedStorage`, `addItemToNestedStorage`, `extractNestedItemToActor`, `deleteNestedItemFromStorage`.
- В `item-sheet.mjs` добавить построение строк nested storage по аналогии с `_buildInventoryDisplayRows` в [`module/sheets/actor-sheet.mjs`](E:/FoundryVTT/Data/systems/spaceholder/module/sheets/actor-sheet.mjs), но scoped только на текущий предмет.
- В [`template.json`](E:/FoundryVTT/Data/systems/spaceholder/template.json) и нормализации [`module/helpers/item-container.mjs`](E:/FoundryVTT/Data/systems/spaceholder/module/helpers/item-container.mjs) расширить `system.container` новыми полями лимитов, например `limits.maxItems` и `limits.maxWeight`, где `0` или пустое значение означает “без лимита”.
- Для обычного `system.container` проверять лимиты перед `moveActorItemIntoContainer`, созданием нового предмета внутри контейнера и world/UUID переносом на актёра. Количество считать по сумме `system.quantity`, вес — по `quantity * weight` с учётом добавляемого предмета.
- Для гибридного nested-storage режима считать занятость по сумме `system.quantity` у вложенных патронов, лимит количества брать из `system.weapon.ammo.capacity`, а весовой лимит применять только если он задан в `system.container.limits.maxWeight`.
- При добавлении учитывать вместимость магазина: не давать положить больше доступного места, а в UI показывать `занято / вместимость`.
- Если задан калибр магазина (`system.weapon.ammo.caliber`), предупреждать или блокировать drop несовместимых патронов по тому же правилу совместимости, что использует оружейный runtime, если это правило доступно без лишнего экспорта. Если нет, сделать локальную простую проверку “пустой калибр подходит ко всему, иначе строка должна совпасть/пересечься”.
