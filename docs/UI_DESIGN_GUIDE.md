# SpaceHolder: Руководство по дизайну интерфейса

Документ описывает предпочтительные паттерны и правила дизайна UI системы SpaceHolder. Используйте его при доработках интерфейса — как справочник для себя и как контекст для AI-агентов.

**Источник:** анализ эталонных экранов + заметки автора (2025).

---

## 1. Принципы дизайна

### 1.1 Минимализм — без лишнего текста

**Правило:** Избегать избыточных заголовков, надписей, подписей и подсказок.

- Если мы во вкладке «Инвентарь» — не нужен ещё и заголовок «Инвентарь» (это уже в названии вкладки).
- Один заголовок секции достаточен; дублирование только путает.
- **Подсказки допустимы**, когда добавляют смысл. Но нельзя перегибать:
  - ❌ Чекбокс «Включение перемещения» + tooltip «Включение перемещения» + подпись «Включает перемещение».
  - ✅ Чекбокс с иконкой + tooltip «Включает режим перемещения на карте» (если нужно уточнение).
  - ✅ Иконка без текста + tooltip с расшифровкой — хороший вариант.

**Эталон:** Характеристики (один заголовок секции), Инвентарь (заголовок + данные без дублирования), Настройки карты (лаконичные лейблы).

### 1.2 Иконки вместо надписей

**Правило:** Предпочитать иконки там, где они универсально понятны и экономят место.

- Иконки короче текста, работают в любом языке.
- Если иконка может быть непонятна — добавляем tooltip.
- **План (Инвентарь):** убрать «Общий вес» и «Предметов», оставить иконки + tooltip с текстом.

**Эталон:** Кнопки-действия в Ивентах (edit, delete, resolve и т.д.), иконки в настройках карты (leaf, circle-half-stroke, eye-slash и т.д.), иконка в заголовке «Характеристики» (fa-dumbbell).

### 1.3 Селекторы вместо выпадающих списков

**Правило:** Использовать группы кнопок (селекторы), а не `<select>` / dropdown, где это уместно.

- Селектор показывает все варианты сразу, не требует раскрытия.
- Удобнее на мобильных и при небольшом числе опций (2–7).

**Эталон:** Настройки карты (templates/global-map/edge-panel.hbs) — Биомы, Высоты, Реки, Поворот подписи, Регионы, Сглаживание (0–4), Анимация появления — все реализованы как группы icon-btn или числовых кнопок.

**Исключение:** Ивенты — фильтры «Фракция», «Статус», «Сортировка» используют dropdown. Это считается менее идеальным вариантом; при рефакторе стоит рассмотреть селекторы.

### 1.4 Акцентные цвета

**Правило:** Использовать акцентные цвета для выделения активного состояния и важных элементов.

- Цвет не обязан быть зелёным — он должен быть контекстно уместен.
- Зелёные заголовки секций (section-header) — один из вариантов, не обязательный для всех секций.
- Акцент помогает обозначить: выбранное значение, активная кнопка, действие по умолчанию, статус (например, orange для «не выбрано», blue для «действие»).

**Эталон:** Анатомия (оранжевый статус «не выбрано», синяя кнопка «Выбрать»), Настройки карты (зелёная подсветка активной кнопки в группе), заголовки Характеристики/Инвентарь (зелёный блок).

### 1.5 Скруглённые блоки и кнопки-иконки

**Правило:** Использовать скруглённые углы для блоков, карточек, кнопок.

- Современный, мягкий визуальный стиль.
- Кнопки-иконки — компактный способ действий; текст — только при необходимости.

**Эталон:** Ивенты (кнопки edit/delete/history, карточки событий), Анатомия (рамка, кнопки), Настройки карты (все элементы), Характеристики (блоки аббревиатур и значений).

## 2. Эталонные экраны и код

### 2.1 Характеристики (Stats)

- **Назначение:** Отображение основных статов (END, STR, DEX, COR, PER, INT, LUC) и модификаторов.
- **Дизайн:** Секция с зелёным заголовком (иконка + текст «Характеристики»), ряды: аббревиатура (скруглённый бейдж) + полное название + значение + модификатор.
- **Код:**  
  - Шаблон: `templates/actor/actor-character-sheet.hbs` (tab stats, `.stats-panel`, `.section-header`, `.capacities-list`, `.capacity-line`).  
  - Стили: SCSS с префиксом `spaceholder-` / `.section-header` и т.п.

### 2.2 Инвентарь (Inventory)

- **Назначение:** Список предметов, статистика (вес, количество), пустое состояние.
- **Дизайн:** Зелёный заголовок с иконкой рюкзака, строки статистики (иконка + текст; планируется замена текста на tooltip), кнопка «+ Новый предмет», пустое состояние — иконка коробки + подсказка.
- **Код:**  
  - Шаблон: `templates/actor/parts/actor-inventory.hbs`.  
  - Локализация: `lang/ru.json` (SPACEHOLDER.Inventory.*).

### 2.3 Ивенты (Events)

- **Назначение:** Управление черновиками и событиями фракций.
- **Дизайн:** Табы (Черновики / События фракций), фильтры (сейчас dropdown — кандидат на селекторы), список событий, панель деталей с кнопками-иконками (reply, resolve, edit, timeline, mark finished, delete).
- **Замечание:** Экран «не до конца нравится», но служит примером кнопок-иконок и скруглённых блоков.
- **Код:**  
  - Шаблон: `templates/events/events-app.hbs`.  
  - Стили: `src/scss/components/_events.scss`, `_journal-check.scss`.

### 2.4 Настройки карты (Global Map Edge UI)

- **Назначение:** Видимость биомов, высот, рек, регионов, сглаживание, анимация появления.
- **Дизайн:** Лейбл слева, справа — группа icon-btn (или числовых кнопок для сглаживания). Активная кнопка — зелёный фон. Все элементы скруглены.
- **Код:**  
  - Шаблон: `templates/global-map/edge-panel.hbs` (flyout справа — `.sh-gm-edge__flyout--right`).  
  - Стили: `src/scss/components/_globalmap-edge-ui.scss`.  
  - Модуль: `module/helpers/global-map/*`.

### 2.5 Анатомия (Health tab)

- **Назначение:** Части тела, выбор/сброс анатомии.
- **Дизайн:** Секция «No body parts data available» при отсутствии данных; блок «Анатомия:» с двумя элементами: статус (оранжевый) и кнопка «+ Выбрать анатомию» (синяя). Пример дифференциации акцентных цветов.
- **Код:**  
  - Шаблон: `templates/actor/parts/actor-health.hbs` (`.anatomy-management-container`, `.current-anatomy`, `.no-anatomy`, `.anatomy-toggle-btn`).  
  - Логика: `module/sheets/actor-sheet.mjs`, `module/anatomy-manager.mjs`.

## 3. Структура стилей

- SCSS: `src/scss/` → `css/spaceholder.css` (после `npm run build`).
- Компоненты: `src/scss/components/_*.scss`.
- Префиксы: `spaceholder-`, `sh-` (например, `sh-events__`, `sh-gm-edge__`).
- Общие классы: `section-header`, `icon-btn`, и т.д. — см. эталонные шаблоны.

---

## 4. Антипаттерны (чего избегать)

1. **Тройное дублирование:** чекбокс + tooltip с тем же текстом + подпись.
2. **Лишние заголовки:** заголовок секции, когда контекст уже ясен из вкладки или соседних элементов.
3. **Dropdown вместо селектора:** когда вариантов мало (2–7) и они фиксированы — лучше группа кнопок.
4. **Текст вместо иконки:** для действий (edit, delete, add) и повторяющихся меток — предпочтительны иконки + tooltip.
5. **Одинаковый акцент везде:** не нужно делать все заголовки только зелёными блоками; использовать другие цвета, где применимо, или не использовать вовсе.

- Документ можно дополнять новыми скриншотами и уточнениями по мере эволюции дизайна.
